Sample Files for Reference

Monoline, our preferred ebook production pipeline, provided a sample set of accessible EPUBs and XHTML snippets to illustrate “born accessible” in practice.

Examples include:

  • Properly structured chapters
  • Semantic headings and lists
  • Navigation landmarks
  • Accessibility metadata in content.opf
  • Optional ONIX sample metadata for distribution feeds

They are hosted openly on GitHub for authors and publishers to reference or adapt.

For more on Monoline’s production approach, see monoline.net. The workflow ensures EPUBs are compliant, structurally sound, and screen-reader friendly—right out of the box.
