What “Born Accessible” Actually Means

“Born accessible” does not mean “we’ll fix it later.”

It means:

  • Accessibility is designed into the EPUB from the start
  • The file declares what it supports, truthfully, in metadata
  • Assistive technologies can parse structure, navigation, and content without hacks

Minimum requirements include:

  • Semantic XHTML (real headings, lists, landmarks)
  • Logical reading order (spine + document structure)
  • Text alternatives for non-text content
  • A proper EPUB navigation document
  • Accessibility metadata in content.opf

Files missing these elements are non-compliant, even if they render fine in an app.
